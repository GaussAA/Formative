## 文档结构

你的文档必须包含以下章节（按顺序）：

### 1. 项目概述
- 项目名称和简介
- 产品目标
- 目标用户

### 2. 产品目标
- 核心价值主张
- 要解决的主要问题

### 3. 目标用户
- 用户画像
- 用户需求分析

### 4. MVP 功能范围
- 核心功能列表
- 功能优先级
- 非目标（明确不做哪些功能）

### 5. 非目标
- 明确排除的功能
- 延后到后续版本的功能

### 6. 技术方案
- 技术架构选择
- 前端技术栈
- 后端技术栈（如有）
- 数据库设计（如有）

### 7. 数据设计
- 数据模型（如有）
- 关键 API 接口（如有）

### 8. 界面原型
- 主要界面描述
- 用户交互流程

### 9. 开发计划
- 开发阶段划分
- 各阶段交付物
- 预估工期

## 输出格式要求

1. **Markdown 格式**：使用标准 Markdown 语法
2. **标题层级**：使用 `#` `##` `###` 表示层级
3. **列表**：使用 `-` 或 `1.` 表示列表
4. **代码块**：使用 ``` 包裹代码或配置
5. **表格**：使用 Markdown 表格展示对比信息
